OS Popup Platform Integration

이 문서는 feat: add os-level popup integration for macos and windows 커밋을 따라가기 위한 문서입니다. 역할은 플랫폼 연결과 배포 구조를 잡는 것입니다.

중요: Windows에서는 taskbar popup이 목표이므로 앱 내부 fallback을 막고, Release 폴더에 scripts/ 폴더 전체를 복사해야 합니다.

수정 파일

  • lib/shared/services/taskbar_popup_service.dart
  • lib/features/notifications/presentation/app_toast.dart
  • macos/Runner/AppDelegate.swift
  • macos/Runner/MainFlutterWindow.swift
  • macos/Runner.xcodeproj/project.pbxproj
  • windows/CMakeLists.txt
  • build_windows.bat
  • required.txt

1. 공통 서비스 추가

lib/shared/services/taskbar_popup_service.dart
class TaskbarPopupService {
  ...

  Future<bool> show(...) async {
    if (Platform.isMacOS) {
      ...
    }

    if (!Platform.isWindows) {
      return false;
    }

    final scriptPath = _findScriptPath();
    if (scriptPath == null) {
      return false;
    }

    await Process.start(
      executable,
      [
        '-NoProfile',
        '-NonInteractive',
        '-WindowStyle', 'Hidden',
        '-ExecutionPolicy', 'Bypass',
        '-File', scriptPath,
        ...
      ],
    );
  }
}

여기서 macOS는 MethodChannel, Windows는 PowerShell 호출로 분기합니다.

2. 스크립트 경로를 Release 실행 기준으로 찾기

lib/shared/services/taskbar_popup_service.dart
addAncestors(Directory.current.path);
addAncestors(File(Platform.resolvedExecutable).parent.path);

이 로직이 있어야 Windows Release 폴더에서 .exe를 직접 실행해도 scripts/show_taskbar_popup.ps1를 찾을 수 있습니다.

3. app_toast.dart를 OS 우선 구조로 바꾸기

lib/features/notifications/presentation/app_toast.dart
void showAppToast(...) {
  unawaited(
    _showTaskbarPopupOrFallback(...),
  );
}
if (!shown && !Platform.isWindows && context.mounted) {
  _ToastOverlay.instance.show(...);
}

Windows에서는 fallback을 막아야 “실패했을 때 앱 안에 토스트가 뜨는 문제”를 피할 수 있습니다.

6. Windows Release에 scripts 폴더 전체 복사

  DIRECTORY "${CMAKE_CURRENT_SOURCE_DIR}/../scripts/"
  DESTINATION "${INSTALL_BUNDLE_LIB_DIR}/scripts"
  COMPONENT Runtime
  FILES_MATCHING
    PATTERN "*.ps1"
    PATTERN "*.svg"
)

7. Windows 빌드 도우미

build_windows.bat

Windows 빌드를 한 번에 수행하는 배치 파일입니다.

required.txt

Flutter, Visual Studio C++, PowerShell 등 사전 요구사항을 정리한 문서입니다.

완료 기준

  • macOS는 MethodChannel 기반 네이티브 팝업을 먼저 시도한다.
  • Windows는 PowerShell 스크립트를 먼저 시도한다.
  • Windows에서는 앱 내부 fallback을 쓰지 않는다.
  • Release 폴더에 scripts/ 폴더가 통째로 복사된다.
정리: 이 커밋은 연결과 배포 준비입니다. 실제 Windows 카드 렌더링은 다음 문서의 두 번째 커밋에서 완성합니다.
