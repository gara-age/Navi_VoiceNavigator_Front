로컬 UI 상태 기록과 설정 동작 규칙 마무리

이 문서는 현재 실제 코드 상태를 기준으로 다시 정리한 Day 10 최종판입니다. 중간 수정 사항까지 포함해서, 지금 프로젝트에 실제로 반영된 내용만 설명합니다.

1. Day 10에서 최종적으로 해결한 문제

문제 1. 앱의 현재 UI 상태를 외부에서 알 수 없음

목표 프로젝트에는 앱 포커스 상태와 설정 모달 열림 상태를 기록하는 계층이 있습니다. 우리 프로젝트에도 그 기반을 추가했습니다.

문제 2. 데모 모드 화면 설정이 상위 테마에 반영되지 않음

원인은 데모 모드를 DemoHomePage로 직접 열고 있었기 때문입니다. 지금은 반드시 VoiceNavigatorDemoApp을 통해 진입하도록 수정되었습니다.

문제 3. 데모 설정이 저장 전에도 적용됨

원인은 onChanged로 상위 상태를 즉시 바꾸고 있었기 때문입니다. 지금은 저장 버튼일 때만 실제 반영되고, 취소/닫기면 draft만 버립니다.

문제 4. 고대비와 다크테마가 동시에 켜지거나 토글 방향이 꼬임

지금은 마지막으로 사용자가 켠 토글이 우선합니다.

  • 다크테마 → 고대비: 고대비가 켜지고 다크테마는 꺼집니다.
  • 고대비 → 다크테마: 다크테마가 켜지고 고대비는 꺼집니다.
  • 큰 글씨는 둘 중 어느 쪽과도 함께 켤 수 있습니다.

2. 새 파일 만들기: local_ui_state_service.dart

lib/shared/services/local_ui_state_service.dart

먼저 새로운 서비스 파일을 만듭니다. 이 파일은 앱이 지금 어떤 UI 상태인지 runtime/ui_state.json에 기록합니다. 

import 'dart:convert';
import 'dart:io';

class LocalUiStateService {
  LocalUiStateService._();

  static final LocalUiStateService instance = LocalUiStateService._();

  Future<void> setSettingsModalOpen(bool isOpen) async {
    await _writeState({
      'settings_modal_open': isOpen,
      'updated_at_ms': DateTime.now().millisecondsSinceEpoch,
    });
  }

  Future<void> setAppFocused(bool isFocused) async {
    await _writeState({
      'app_focused': isFocused,
      'updated_at_ms': DateTime.now().millisecondsSinceEpoch,
    });
  }

  Future<void> _writeState(Map<String, dynamic> patch) async {
    final file = await _resolveStateFile();
    if (file == null) {
      return;
    }

    try {
      final current = await _readState(file);
      current.addAll(patch);
      await file.parent.create(recursive: true);
      await file.writeAsString(jsonEncode(current));
    } catch (_) {
      // Ignore local state write failures.
    }
  }

  Future<Map<String, dynamic>> _readState(File file) async {
    if (!await file.exists()) {
      return <String, dynamic>{};
    }

    try {
      final raw = await file.readAsString();
      return Map<String, dynamic>.from(
        jsonDecode(raw) as Map<String, dynamic>,
      );
    } catch (_) {
      return <String, dynamic>{};
    }
  }

  Future<File?> _resolveStateFile() async {
    final envRoot = Platform.environment['VOICE_NAVIGATOR_ROOT'];
    if (envRoot != null && envRoot.isNotEmpty) {
      return File(
        '$envRoot${Platform.pathSeparator}runtime${Platform.pathSeparator}ui_state.json',
      );
    }

    final roots = <Directory>[];

    void addAncestors(Directory directory) {
      var current = directory;
      for (var i = 0; i < 8; i++) {
        roots.add(current);
        final parent = current.parent;
        if (parent.path == current.path) {
          break;
        }
        current = parent;
      }
    }

    addAncestors(Directory.current);
    addAncestors(File(Platform.resolvedExecutable).parent);

    for (final root in roots) {
      final runtimeDir = Directory(
        '${root.path}${Platform.pathSeparator}runtime',
      );
      if (runtimeDir.existsSync()) {
        return File(
          '${runtimeDir.path}${Platform.pathSeparator}ui_state.json',
        );
      }
    }

    return null;
  }
}
이 파일이 기록하는 값
현재 기준으로는 app_focused, settings_modal_open, updated_at_ms 세 값을 기록합니다.

실제로 중요한 최종 로직은 이 메서드입니다. 새로 켠 토글을 우선 적용하고, 반대쪽 토글은 자동으로 끕니다. 

void _updateDisplay({
  bool? darkTheme,
  bool? highContrast,
  bool? largeText,
}) {
  final current = _draftSettings.display;
  var nextDarkTheme = darkTheme ?? current.darkTheme;
  var nextHighContrast = highContrast ?? current.highContrast;
  final nextLargeText = largeText ?? current.largeText;

  if (darkTheme == true) {
    nextHighContrast = false;
  }
  if (highContrast == true) {
    nextDarkTheme = false;
  }

  _update(
    _draftSettings.copyWith(
      display: current.copyWith(
        darkTheme: nextDarkTheme,
        highContrast: nextHighContrast,
        largeText: nextLargeText,
      ),
    ),
  );
}

토글 연결 방식 변경

onChanged: (value) => _updateDisplay(darkTheme: value)
onChanged: (value) => _updateDisplay(highContrast: value)
onChanged: (value) => _updateDisplay(largeText: value)

9. 실행 후 꼭 확인할 것

  1. 앱 실행 후 runtime/ui_state.json 파일이 생성되는지 확인합니다.
  2. 앱에 포커스가 있을 때 app_focused: true인지 확인합니다.
  3. 실제/데모 설정 모달을 열면 settings_modal_open: true로 바뀌는지 확인합니다.
  4. 설정 모달을 닫으면 settings_modal_open: false로 돌아오는지 확인합니다.
  5. 데모 모드에서 화면 설정을 바꾼 뒤 저장을 눌렀을 때만 반영되는지 확인합니다.
  6. 데모 모드에서 취소/닫기로 나오면 변경이 버려지는지 확인합니다.
  7. 다크테마 → 고대비, 고대비 → 다크테마 전환이 둘 다 되는지 확인합니다.
  8. 하지만 둘이 동시에 켜지지는 않는지 확인합니다.
