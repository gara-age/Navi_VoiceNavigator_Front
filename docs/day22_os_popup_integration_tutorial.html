Day 22

OS Popup Integration

22일차는 기존 앱 내부 토스트 구조를 유지하되, 표시 우선순위를 운영체제 팝업 쪽으로 바꾸는 작업입니다. 현재 최종 구조는 macOS는 MethodChannel + NSPanel, Windows는 PowerShell + WPF, 그리고 Flutter fallback은 비-Windows 환경에서만 쓰는 형태입니다.

핵심: Windows는 taskbar popup처럼 보여야 하므로 앱 내부 fallback을 막고, Release 폴더에 scripts/와 SVG 아이콘이 실제로 복사되게 만들어야 합니다.

현재 최종 구조

  1. showAppToast(...) 호출
  2. TaskbarPopupService.instance.show(...)로 OS 팝업 시도
  3. macOS면 voice_navigator/taskbar_popup MethodChannel 호출
  4. Windows면 scripts/show_taskbar_popup.ps1 실행
  5. OS 팝업 실패 시 Flutter fallback은 비-Windows에서만 사용

수정 파일

  • lib/shared/services/taskbar_popup_service.dart
  • lib/features/notifications/presentation/app_toast.dart
  • macos/Runner/AppDelegate.swift
  • macos/Runner/MainFlutterWindow.swift
  • macos/Runner.xcodeproj/project.pbxproj
  • windows/CMakeLists.txt
  • build_windows.bat
  • required.txt
  • scripts/show_taskbar_popup.ps1

1. Flutter 쪽 공통 서비스

lib/shared/services/taskbar_popup_service.dart
Future<bool> show({
  required String title,
  required String message,
  required TaskbarPopupState state,
  required TaskbarPopupThemeMode themeMode,
  required bool largeText,
  int durationMs = 5000,
}) async {
  if (Platform.isMacOS) {
    ...
  }

  if (!Platform.isWindows) {
    return false;
  }

  final scriptPath = _findScriptPath();
  if (scriptPath == null) {
    return false;
  }

  await Process.start(
    executable,
    [
      '-NoProfile',
      '-NonInteractive',
      '-WindowStyle', 'Hidden',
      '-ExecutionPolicy', 'Bypass',
      '-File', scriptPath,
      ...
    ],
  );
}

여기서 중요한 부분은 PowerShell 콘솔 창이 튀지 않도록 숨김 옵션을 넣는 것과, 스크립트 경로를 현재 폴더뿐 아니라 실행 파일 상위 디렉터리까지 찾는 것입니다.

2. Windows fallback 차단

lib/features/notifications/presentation/app_toast.dart
if (!shown && !Platform.isWindows && context.mounted) {
  _ToastOverlay.instance.show(
    context,
    title: title,
    message: message,
    state: state,
    displaySettings: displaySettings,
    durationMs: durationMs,
  );
}

이 조건이 핵심입니다. Windows에서 taskbar popup 호출이 실패해도 앱 안쪽 토스트가 뜨면 목표 프로젝트와 다른 동작이 되기 때문에, Windows에서는 fallback을 막습니다.

5. Windows Release에 scripts 폴더 전체 복사

windows/CMakeLists.txt
install(
  DIRECTORY "${CMAKE_CURRENT_SOURCE_DIR}/../scripts/"
  DESTINATION "${INSTALL_BUNDLE_LIB_DIR}/scripts"
  COMPONENT Runtime
  FILES_MATCHING
    PATTERN "*.ps1"
    PATTERN "*.svg"
)

지금 구조에서는 show_taskbar_popup.ps1뿐 아니라 popup_icons/*.svg도 필요하기 때문에, scripts 전체를 복사해야 합니다.

6. Windows 빌드 보조 파일

build_windows.bat

Windows에서 바로 빌드할 수 있도록 Flutter 설정, pub get, Release 빌드까지 묶었습니다.

required.txt

Visual Studio C++ workload, Flutter, PowerShell, PATH 등 실행 전 필요한 항목을 정리했습니다.

7. 최종 크기와 보정값

  • 기본 카드: 320 x 96
  • 큰 글씨 카드: 372 x 118
  • 큰 글씨 제목/본문: 17 / 15
  • Flutter fallback 세로 패딩: 기본 10, 큰 글씨 12

완료 기준

  • macOS는 MethodChannel 기반 네이티브 팝업이 먼저 시도된다.
  • Windows는 PowerShell + WPF 팝업이 우하단에 뜬다.
  • Windows에서는 앱 내부 fallback을 띄우지 않는다.
  • Release 폴더에 scripts/show_taskbar_popup.ps1와 scripts/popup_icons/*.svg가 함께 복사된다.
