Day 20. Widget Exact Sync

20일차는 공통 위젯을 목표 프로젝트와 더 정확히 맞추는 단계입니다. 이번 정리는 초기에 적용한 구조 변경만이 아니라, 이후 스크린샷 비교 과정에서 발견된 보정 사항까지 포함한 최종 상태 기준 문서입니다.

1. title_bar.dart: 플랫폼 분기

lib/features/home/presentation/widgets/title_bar.dart

Windows 스타일과 macOS 스타일은 의도적으로 다르게 가져갑니다. 따라서 이 파일은 “하나의 디자인으로 통일”하는 대신, 플랫폼에 따라 다른 위젯을 반환하는 분기용 셸이 됩니다.

import 'dart:io';

class AppTitleBar extends StatelessWidget {
  const AppTitleBar({super.key});

  @override
  Widget build(BuildContext context) {
    if (Platform.isMacOS) {
      return const AppTitleBarMacos();
    }

    return Container(
      height: 64,
      ...
    );
  }
}

즉 Windows에서는 기존 인앱 타이틀바를 쓰고, macOS에서는 별도 파일의 macOS 스타일 타이틀바를 씁니다.

2. title_bar_macos.dart: macOS 전용 타이틀바

lib/features/home/presentation/widgets/title_bar_macos.dart

목표 프로젝트의 macOS 빌드처럼, 우측 정렬 텍스트와 우측 마이크 아이콘을 사용하는 미니멀한 구조를 별도 파일로 구현합니다.

class AppTitleBarMacos extends StatelessWidget {
  const AppTitleBarMacos({super.key, this.onOpenHelp});

  final VoidCallback? onOpenHelp;

  @override
  Widget build(BuildContext context) {
    final surfaceTheme = Theme.of(context).extension<AppSurfaceTheme>()!;

    return Column(
      mainAxisSize: MainAxisSize.min,
      children: [
        Container(
          height: 56,
          padding: const EdgeInsets.symmetric(horizontal: 20),
          decoration: BoxDecoration(
            color: surfaceTheme.surface,
            border: Border(
              bottom: BorderSide(color: surfaceTheme.border),
            ),
          ),
          child: Row(
            children: [
              Expanded(
                child: Column(
                  mainAxisAlignment: MainAxisAlignment.center,
                  crossAxisAlignment: CrossAxisAlignment.end,
                  children: [
                    Text('Navi: Voice Navigator', textAlign: TextAlign.right),
                    Text(
                      'AI Voice Assistant for PC Accessibility',
                      textAlign: TextAlign.right,
                    ),
                  ],
                ),
              ),
              const SizedBox(width: 12),
              Container(
                width: 36,
                height: 36,
                child: const Icon(Icons.mic_rounded),
              ),
            ],
          ),
        ),
      ],
    );
  }
}

3. macOS 시스템 타이틀바 숨기기

macos/Runner/MainFlutterWindow.swift

Flutter 위젯 안의 macOS 전용 타이틀바를 보여주려면, 시스템 기본 title bar를 숨겨야 합니다. 이 보정은 Day 20 초기 문서에는 없었지만 실제 구현에서는 중요합니다.

override func awakeFromNib() {
  let flutterViewController = FlutterViewController()
  let windowFrame = self.frame
  self.contentViewController = flutterViewController
  self.setFrame(windowFrame, display: true)
  self.titleVisibility = .hidden
  self.titlebarAppearsTransparent = true
  self.isMovableByWindowBackground = true
  self.styleMask.insert(.fullSizeContentView)
  self.toolbar = nil

  RegisterGeneratedPlugins(registry: flutterViewController)

  super.awakeFromNib()
}

이 설정으로 macOS에서는 시스템 title 영역이 사라지고, 앱 내부의 AppTitleBarMacos만 보이게 됩니다.

4. ready_state.dart: 목표형 scroll-safe 구조 채택

lib/features/home/presentation/widgets/ready_state.dart

문구는 현재 버전을 유지하되, 레이아웃은 목표 파일의 LayoutBuilder + SingleChildScrollView + ConstrainedBox 구조를 채택합니다. 덕분에 창 높이가 작아졌을 때 overflow가 줄고, 중앙 정렬 느낌도 유지됩니다.

return Container(
  color: surfaceTheme.surface,
  child: LayoutBuilder(
    builder: (context, constraints) {
      return SingleChildScrollView(
        padding: const EdgeInsets.all(32),
        child: ConstrainedBox(
          constraints: BoxConstraints(minHeight: constraints.maxHeight - 64),
          child: Center(
            child: ConstrainedBox(
              constraints: const BoxConstraints(maxWidth: 560),
              child: Column(
                mainAxisAlignment: MainAxisAlignment.center,
                children: [
                  ...
                ],
              ),
            ),
          ),
        ),
      );
    },
  ),
);

이번 보정에서는 구조만 바뀐 것이 아닙니다. 실제 코드 기준으로는 아래 세 가지도 함께 바뀌었습니다.

  • 바깥 배경이 surfaceTheme.contentBackground가 아니라 surfaceTheme.surface로 바뀌었습니다.
  • busy 상태 아이콘 색이 기본 muted가 아니라 surfaceTheme.accent를 쓰도록 바뀌었습니다.
  • 요약 본문이 fontWeight: FontWeight.w700로 강화됐습니다.

그리고 스크린샷 보정으로, 결과 요약 아래 실제 요약 본문은 bold로 표시되도록 정리했습니다.

Text(
  summary!,
  style: TextStyle(
    fontSize: 14,
    fontWeight: FontWeight.w700,
    height: 1.6,
    color: surfaceTheme.textPrimary,
  ),
)

다음 제안 라벨과 현재 제품형 문구는 그대로 유지합니다.

5. action_panel.dart: 목표 화면형으로 다시 정렬

lib/features/home/presentation/widgets/action_panel.dart

Day 19에서 공통 제품 톤으로 다듬었던 일부 카피는, 목표 프로젝트와 완전 동일하게 맞추는 기준에서는 다시 목표형으로 되돌릴 필요가 있었습니다. 따라서 이번 단계에서는 버튼 설명, shortcut 칩 모양, 모드 전환 카드의 형태를 목표 화면 기준으로 다시 맞춥니다.

_CommandButton(
  icon: Icons.mic_rounded,
  title: isRecording ? '듣는 중' : '음성 듣기',
  description: '음성 명령 인식을 시작합니다.',
  shortcut: ShortcutUtils.displayLabel(listenShortcut),
  ...
)
_CommandButton(
  icon: Icons.visibility_rounded,
  title: '현재 화면 읽기',
  description: '현재 화면 요약과 읽기를 시작합니다.',
  ...
)
_CommandButton(
  icon: Icons.settings_rounded,
  title: '설정',
  description: '기본 설정, 단축키, 보안, 화면 모드',
  ...
)

shortcut 칩 radius도 목표처럼 작은 pill이 아니라 각진 짧은 칩으로 정리합니다.

borderRadius: BorderRadius.circular(6)

그리고 모드 전환은 큰 선택 카드 대신, 작은 점 표시 중심의 목표형 구조로 되돌립니다. 여기에는 아래 변경도 포함됩니다.

  • 선택 아이콘 check 제거
  • 좌측 작은 dot 표시 복귀
  • selected 배경색과 border 색을 목표형으로 조정
  • 설명 문구를 Day 19 공통 UX 버전에서 목표 버전으로 되돌림

6. status_card.dart: 목표형 표현 유지

lib/features/home/presentation/widgets/status_card.dart

이 파일은 이미 목표 프로젝트와 큰 방향이 비슷했기 때문에, Day 20에서는 wave, dot, 아이콘 박스, spacing을 목표형 표현에 맞춰 정리하는 수준으로 마무리합니다.

Container(
  width: 40,
  height: 40,
  decoration: BoxDecoration(
    color: iconBackground,
    borderRadius: BorderRadius.circular(10),
  ),
  child: Icon(icon, size: 20, color: iconColor),
)

현재 워크트리 기준 diff는 대부분 포맷 정리에 가깝지만, 문서에는 이 파일이 이번 exact sync 범위에 포함된다는 점을 명확히 남겨둡니다. 즉 StatusCard는 기능 추가보다 목표형 표현 유지와 정렬이 목적입니다.

7. 좌측 ActionPanel stretch 보정

lib/features/home/presentation/home_page.dart
lib/demo/demo_home_page.dart

스크린샷 비교 과정에서, 좌측 패널 위아래에 회색 공간이 비치는 문제가 발견됐습니다. 원인은 본문 Row가 세로로 stretch되지 않아서 왼쪽 패널이 내용 높이만큼만 렌더링되던 점이었습니다.

Expanded(
  child: Row(
    crossAxisAlignment: CrossAxisAlignment.stretch,
    children: [
      ...
    ],
  ),
)

이 보정으로 실제 모드와 데모 모드 모두 왼쪽 패널이 본문 높이를 안정적으로 채우게 됩니다.

8. 상단 Status Bar 하단선 복구

lib/features/home/presentation/home_page.dart

상단 상태 카드 영역 아래 구분선이 보이지 않는 문제도 함께 보정했습니다. 상태 바 컨테이너의 단순 color 지정이 아니라, BoxDecoration에 bottom border를 다시 넣어서 목표 화면처럼 상단과 본문이 명확히 구분되도록 했습니다.

Container(
  height: 92,
  decoration: BoxDecoration(
    color: surfaceTheme.surface,
    border: Border(
      bottom: BorderSide(color: surfaceTheme.border),
    ),
  ),
  child: Row(...),
)

이번 단계에서 꼭 이해해야 하는 것

  • title_bar는 완전 동일화보다 플랫폼 분기가 맞는 영역입니다.
  • macOS 인앱 타이틀바를 제대로 쓰려면 Swift 쪽 시스템 title bar 숨김도 함께 필요합니다.
  • ready_state는 카피와 레이아웃을 분리해서 다루는 것이 안전합니다.
  • action_panel은 공통 UX 개선보다 목표 화면 exact sync를 우선해야 합니다.
  • 스크린샷 비교 후 발견된 stretch, 구분선, bold 처리 같은 후속 보정도 최종 문서에 포함해야 실제 상태와 맞습니다.

Day 20 완료 기준

  • Windows/macOS에서 인앱 title bar가 의도대로 분기된다.
  • macOS에서 시스템 title bar가 숨겨지고 앱 내부 title bar만 사용된다.
  • ready_state.dart가 목표형 scroll-safe 구조를 가진다.
  • 결과 요약 본문이 더 명확하게 보이도록 bold 처리되었다.
  • action_panel.dart가 목표 화면형 구조와 카피에 더 가까워졌다.
  • 좌측 패널 stretch와 상단 상태 바 하단선까지 포함해 화면 보정이 완료되었다.
