Day 9

앱 실행 진입점 분리와 데모 전용 앱 셸 만들기

이 문서는 Day 8이 끝난 상태에서 시작합니다. 오늘 목표는 기능 하나를 더 넣는 것이 아니라 프로젝트 실행 구조를 목표 프로젝트처럼 정리하는 것입니다.

즉 main.dart 하나로만 앱을 실행하는 구조에서 벗어나, 데모 전용 실행과 실제 모드 전용 실행을 분리하는 단계입니다.

오늘 수정할 파일

수정 파일
lib/app/app_shell.dart

루트 앱이 시작 화면을 외부에서 주입받을 수 있게 만듭니다.

새 파일

데모 모드 전용 실행 진입점입니다.

새 파일

실제 모드 전용 실행 진입점입니다.

새 파일
lib/demo/demo_app.dart

데모 모드 전체를 감싸는 전용 앱 셸입니다.

1. app_shell.dart 수정

lib/app/app_shell.dart

현재 VoiceNavigatorApp은 무조건 ModeLauncherPage에서 시작합니다. 하지만 목표 프로젝트처럼 가려면, 원하는 홈 화면을 바깥에서 주입할 수 있어야 합니다.

수정 1: 생성자에 home 추가

기존 생성자 부분을 아래처럼 바꿉니다.

class VoiceNavigatorApp extends ConsumerStatefulWidget {
  const VoiceNavigatorApp({
    super.key,
    this.home,
  });

  final Widget? home;

수정 2: build 안의 home 변경

기존 home: _loaded ? const ModeLauncherPage() ... 부분을 아래로 바꿉니다.

home: _loaded
    ? (widget.home ?? const ModeLauncherPage())
    : const Scaffold(
        body: Center(
          child: SizedBox(
            width: 28,
            height: 28,
            child: CircularProgressIndicator(strokeWidth: 2.8),
          ),
        ),
      ),
이 수정의 의미
이제 루트 앱은 기본적으로는 모드 선택 화면으로 시작하지만, 원하면 HomePage나 다른 화면을 시작점으로 직접 넣을 수 있습니다.

2. main_demo.dart 새로 만들기

lib/main_demo.dart

이 파일은 데모 모드만 바로 실행하기 위한 전용 entry point입니다. 

import 'package:flutter/material.dart';

import 'demo/demo_app.dart';

void main() {
  WidgetsFlutterBinding.ensureInitialized();
  runApp(const VoiceNavigatorDemoApp());
}

이제 flutter run -t lib/main_demo.dart처럼 실행하면 데모 모드만 바로 띄울 수 있습니다.

3. main_connected.dart 새로 만들기

lib/main_connected.dart

이 파일은 실제 모드만 바로 실행하기 위한 전용 entry point입니다. 

import 'package:flutter/material.dart';

이제 flutter run -t lib/main_connected.dart처럼 실행하면 모드 선택 없이 바로 실제 모드로 들어갑니다.

4. demo_app.dart 새로 만들기

lib/demo/demo_app.dart

이제 데모 모드도 하나의 독립 앱처럼 다뤄야 합니다. 그래서 데모 전용 앱 셸을 만듭니다. 

import 'package:flutter/material.dart';

import '../app/theme/app_theme.dart';
import '../shared/models/settings_models.dart';
import '../shared/services/local_settings_store.dart';
import 'demo_home_page.dart';

class VoiceNavigatorDemoApp extends StatefulWidget {
  const VoiceNavigatorDemoApp({super.key});

  @override
  State<VoiceNavigatorDemoApp> createState() => _VoiceNavigatorDemoAppState();
}

class _VoiceNavigatorDemoAppState extends State<VoiceNavigatorDemoApp> {
  AppSettings _settings = AppSettings.defaults();
  bool _settingsLoaded = false;

  @override
  void initState() {
    super.initState();
    _loadSettings();
  }

  Future<void> _loadSettings() async {
    final loaded = await LocalSettingsStore.instance.load();
    if (!mounted) {
      return;
    }
    setState(() {
      _settings = loaded;
      _settingsLoaded = true;
    });
  }

  Future<void> _saveSettings(AppSettings settings) async {
    await LocalSettingsStore.instance.save(settings);
  }

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'Navi: Voice Navigator Demo',
      debugShowCheckedModeBanner: false,
      theme: buildAppTheme(display: _settings.display),
      builder: (context, child) {
        final mediaQuery = MediaQuery.of(context);
        final scale = _settings.display.largeText ? 1.32 : 1.0;

        return MediaQuery(
          data: mediaQuery.copyWith(
            textScaler: TextScaler.linear(scale),
          ),
          child: DefaultTextStyle.merge(
            style: const TextStyle(fontFamily: 'Pretendard'),
            child: child ?? const SizedBox.shrink(),
          ),
        );
      },
      home: _settingsLoaded
          ? DemoHomePage(
              initialSettings: _settings,
              onSettingsChanged: (settings) {
                setState(() => _settings = settings);
              },
              onSettingsSaved: (settings) {
                setState(() => _settings = settings);
                _saveSettings(settings);
              },
            )
          : const _DemoBootstrapScreen(),
    );
  }
}

class _DemoBootstrapScreen extends StatelessWidget {
  const _DemoBootstrapScreen();

  @override
  Widget build(BuildContext context) {
    return const Scaffold(
      body: Center(
        child: SizedBox(
          width: 28,
          height: 28,
          child: CircularProgressIndicator(strokeWidth: 2.8),
        ),
      ),
    );
  }
}
이 파일의 핵심 역할
데모 모드 설정을 로드하고, 테마를 적용하고, 저장까지 담당하는 데모 전용 루트 앱입니다.

5. demo_home_page.dart 수정

lib/demo/demo_home_page.dart

이제 DemoHomePage는 더 이상 내부에서 기본 설정을 직접 만들지 않습니다. 대신 바깥 demo_app.dart가 전달한 설정을 사용합니다.

수정 1: 생성자와 필드 추가

class DemoHomePage extends StatefulWidget {
  const DemoHomePage({
    super.key,
    required this.initialSettings,
    required this.onSettingsChanged,
    required this.onSettingsSaved,
  });

  final AppSettings initialSettings;
  final ValueChanged<AppSettings> onSettingsChanged;
  final ValueChanged<AppSettings> onSettingsSaved;

수정 2: initState 변경

기존:

_settings = AppSettings.defaults();

변경:

_settings = widget.initialSettings;

수정 3: 설정 저장/변경 콜백 연결

DemoSettingsModal를 여는 부분에서 onSaved / onChanged 안에 부모 콜백을 추가합니다. 

onSaved: (next) {
  setState(() {
    _settings = next;
    _showSettingsModal = false;
  });
  widget.onSettingsSaved(next);
},
onChanged: (next) {
  setState(() => _settings = next);
  widget.onSettingsChanged(next);
},

이 수정으로 이제 데모 홈은 “설정 사용 화면”이 되고, 저장/로드 책임은 바깥 앱 셸이 담당하게 됩니다.

6. main.dart는 왜 그대로 두나

lib/main.dart

기본 실행은 여전히 전체 앱 진입점으로 남겨둡니다. 즉 main.dart는 유지하고, 데모/실제 모드 전용 실행 파일만 추가하는 방식입니다. 

import 'package:flutter/material.dart';
import 'package:flutter_riverpod/flutter_riverpod.dart';

import 'app/app_shell.dart';

void main() {
  WidgetsFlutterBinding.ensureInitialized();
  runApp(const ProviderScope(child: VoiceNavigatorApp()));
}

7. 실행 후 확인할 것

  1. flutter run -t lib/main.dart로 전체 앱이 기존처럼 실행되는지 확인합니다.
  2. flutter run -t lib/main_demo.dart로 데모 모드만 바로 뜨는지 확인합니다.
  3. flutter run -t lib/main_connected.dart로 실제 모드가 바로 뜨는지 확인합니다.
  4. 데모 모드에서 설정을 저장한 뒤 앱을 다시 실행해도 값이 유지되는지 확인합니다.
  5. 데모 모드 설정 변경이 즉시 테마에 반영되는지 확인합니다.

8. 오늘 꼭 이해해야 하는 것

  • 왜 entry point를 여러 개로 나누는가
  • 왜 데모 모드도 별도 앱 셸이 필요한가
  • 왜 DemoHomePage가 설정을 직접 만들지 않고 외부에서 받아야 하는가
  • 왜 VoiceNavigatorApp이 home을 주입받는 구조가 유연한가

9. Day 9 완료 기준

  • main_demo.dart로 데모 모드만 바로 실행 가능하다.
  • main_connected.dart로 실제 모드만 바로 실행 가능하다.
  • demo_app.dart가 데모 전용 앱 셸 역할을 한다.
  • DemoHomePage가 외부에서 설정을 주입받아 사용한다.
  • 프로젝트 실행 구조가 목표 프로젝트와 훨씬 가까워진다.
