Day 5 Tutorial

Flutter 5일차: 실제 모드 UI 완성도 끌어올리기

이 문서는 5일차에 진행한 home_page.dart와 관련 위젯들을 다듬는 작업을 튜토리얼 방식으로 다시 읽을 수 있게 정리한 문서입니다.

1단계

상단 타이틀 바를 더 제품형 UI처럼 보이게 다듬습니다.

2단계

상태 카드의 정보 위계를 더 선명하게 만듭니다.

3단계

액션 패널을 실행 가능한 카드처럼 보이게 개선합니다.

4단계

중앙 결과 영역의 준비 상태와 결과 요약을 더 읽기 좋게 정리합니다.

5단계

마지막으로 HomePage 전체 배경과 레이아웃을 정리합니다.

오늘의 목표

  • 실제 모드 화면이 더 이상 프로토타입처럼 보이지 않아야 합니다.
  • 상단 상태 영역, 왼쪽 액션 패널, 중앙 결과 영역이 더 제품 화면처럼 보여야 합니다.
  • 이미 있는 기능은 그대로 유지하면서 UI 완성도를 높여야 합니다.
  • 데모 모드와 실제 모드가 같은 디자인 시스템 위에 있다는 인상을 주어야 합니다.

먼저 이해할 개념

레이아웃 층위

shell 배경, surface 카드, content 배경처럼 화면의 층을 나누는 방식입니다.

시각적 위계

무엇이 제목이고 무엇이 상태값인지, 어떤 정보가 더 중요한지 눈에 보이게 만드는 방식입니다.

ThemeExtension 활용

1일차에 만든 AppSurfaceTheme를 실제 UI 위젯에서 꺼내 쓰는 과정입니다.

컴포넌트 재구성

위젯을 버리지 않고, 같은 기능을 유지한 채 모양과 정보 구조를 다듬는 작업입니다.

정보 카드 디자인

텍스트만 나열하는 대신 아이콘, 배경, 라벨, 강조색으로 상태를 보여주는 방식입니다.

1. title_bar.dart 개선

lib/features/home/presentation/widgets/title_bar.dart

상단 바는 실제 기능보다 제품 인상을 크게 좌우합니다. 5일차에는 테마의 surface, border, textPrimary, textMuted를 활용해 더 정돈된 헤더로 개선합니다.

핵심 생각: 상단 바는 단순 제목 텍스트가 아니라 “앱 쉘”처럼 보여야 합니다.

2. status_card.dart 개선

lib/features/home/presentation/widgets/status_card.dart

상태 카드는 단순 값 표시를 넘어서, 어떤 정보가 라벨이고 어떤 정보가 핵심 값인지 한눈에 구분돼야 합니다.

  • 아이콘은 별도 배경 박스 안에 넣습니다.
  • 라벨은 작고 연하게, 값은 더 굵고 또렷하게 만듭니다.
  • 전체 카드의 밀도를 높여 상단 영역이 더 제품답게 보이게 합니다.

3. action_panel.dart 개선

lib/features/home/presentation/widgets/action_panel.dart

왼쪽 액션 패널은 사용자가 가장 자주 상호작용하는 영역입니다. 그래서 단순 버튼보다 “실행 가능한 기능 카드”처럼 보여야 합니다.

  • 패널 상단에 짧은 설명 문구를 추가합니다.
  • 각 버튼에 아이콘, 제목, 설명을 넣어 카드형 액션으로 만듭니다.
  • 테마에서 가져온 색을 사용해 다크/고대비 대응이 쉬운 구조로 갑니다.

4. ready_state.dart 개선

lib/features/home/presentation/widgets/ready_state.dart

중앙 결과 영역은 실제 모드의 핵심입니다. 결과가 없을 때도 어색하지 않아야 하고, 결과가 생기면 더 읽기 좋게 보여야 합니다.

  • 준비 상태/처리 상태 안내 문구를 분명히 나눕니다.
  • 결과 요약 카드를 더 선명한 박스로 만듭니다.
  • 후속 질문은 결과 카드 안쪽 보조 박스로 다시 분리해 정보 위계를 만듭니다.

최종 코드 5: home_page.dart

lib/features/home/presentation/home_page.dart
import 'package:flutter/material.dart';
import 'package:flutter_riverpod/flutter_riverpod.dart';

import '../../../../app/theme/app_theme.dart';
import '../../listening/application/listening_controller.dart';
import '../../session/application/session_controller.dart';
import '../../settings/application/settings_controller.dart';
import '../../settings/presentation/settings_modal.dart';
import 'widgets/action_panel.dart';
import 'widgets/ready_state.dart';
import 'widgets/status_card.dart';
import 'widgets/title_bar.dart';

class HomePage extends ConsumerStatefulWidget {
  const HomePage({super.key});

  @override
  ConsumerState<HomePage> createState() => _HomePageState();
}

class _HomePageState extends ConsumerState<HomePage> {
  bool _showSettings = false;

  Future<void> _handleListen() async {
    final session = ref.read(sessionControllerProvider);
    final sessionNotifier = ref.read(sessionControllerProvider.notifier);
    final listeningNotifier = ref.read(listeningControllerProvider.notifier);

    if (!session.isRecording) {
      listeningNotifier.startListening();
      await sessionNotifier.startListening();
      return;
    }

    listeningNotifier.setProcessing();
    await sessionNotifier.stopListeningAndProcess();
    listeningNotifier.reset();
  }

  Future<void> _handleScreenRead() async {
    final listeningNotifier = ref.read(listeningControllerProvider.notifier);
    final sessionNotifier = ref.read(sessionControllerProvider.notifier);

    listeningNotifier.setProcessing();
    await sessionNotifier.triggerScreenRead();
    listeningNotifier.reset();
  }

  Future<void> _handleSubmitText(String text) async {
    final listeningNotifier = ref.read(listeningControllerProvider.notifier);
    final sessionNotifier = ref.read(sessionControllerProvider.notifier);

    listeningNotifier.setProcessing();
    await sessionNotifier.submitTextCommand(text);
    listeningNotifier.reset();
  }

  String _micLabel(ListeningState state) {
    switch (state.status) {
      case ListeningStatus.idle:
        return '대기';
      case ListeningStatus.listening:
        return '듣는 중';
      case ListeningStatus.processing:
        return '처리중';
    }
  }

  @override
  Widget build(BuildContext context) {
    final listeningState = ref.watch(listeningControllerProvider);
    final sessionState = ref.watch(sessionControllerProvider);
    final settings = ref.watch(settingsControllerProvider);
    final surfaceTheme = Theme.of(context).extension<AppSurfaceTheme>()!;

    return Scaffold(
      backgroundColor: surfaceTheme.shellBackground,
      body: SafeArea(
        child: Stack(
          children: [
            Column(
              children: [
                const AppTitleBar(),
                Container(
                  height: 92,
                  color: surfaceTheme.surface,
                  child: Row(
                    children: [
                      Expanded(
                        child: StatusCard(
                          label: '마이크 상태',
                          value: _micLabel(listeningState),
                          icon: Icons.mic_none_rounded,
                        ),
                      ),
                      VerticalDivider(
                        width: 1,
                        thickness: 1,
                        color: surfaceTheme.border,
                      ),
                      Expanded(
                        child: StatusCard(
                          label: '현재 모드',
                          value: settings.security.secureInputMode
                              ? '보안 입력 모드'
                              : '일반 모드',
                          icon: settings.security.secureInputMode
                              ? Icons.lock_outline_rounded
                              : Icons.volume_up_outlined,
                        ),
                      ),
                    ],
                  ),
                ),
                Expanded(
                  child: Row(
                    children: [
                      SizedBox(
                        width: 290,
                        child: ActionPanel(
                          isRecording: sessionState.isRecording,
                          onListenPressed: _handleListen,
                          onScreenReadPressed: _handleScreenRead,
                          onSettingsPressed: () {
                            setState(() => _showSettings = true);
                          },
                        ),
                      ),
                      Expanded(
                        child: ReadyState(
                          summary: sessionState.lastSummary,
                          followUp: sessionState.lastFollowUp,
                          isBusy: sessionState.isBusy,
                          onSubmitText: _handleSubmitText,
                        ),
                      ),
                    ],
                  ),
                ),
              ],
            ),
            if (_showSettings) ...[
              Positioned.fill(
                child: GestureDetector(
                  onTap: () => setState(() => _showSettings = false),
                  child: Container(
                    color: const Color(0x66000000),
                  ),
                ),
              ),
              Positioned.fill(
                child: SettingsModal(
                  onClose: () {
                    setState(() => _showSettings = false);
                  },
                ),
              ),
            ],
          ],
        ),
      ),
    );
  }
}

  • 실제 모드 화면이 더 이상 프로토타입처럼 보이지 않습니다.
  • 상단 상태 카드, 왼쪽 패널, 중앙 결과 영역의 층위가 분명해집니다.

동작 기준

  • 기존 듣기/화면 읽기/텍스트 명령 기능은 그대로 유지됩니다.
  • 설정 모달 오버레이와 상태 반영이 자연스럽게 이어집니다.

학습 기준

  • ThemeExtension를 실제 화면에서 어떻게 쓰는지 설명할 수 있습니다.
  • 왜 작은 위젯부터 다듬고 마지막에 HomePage를 맞추는지 이해합니다.
  • 결과 영역에서 정보 위계를 어떻게 만드는지 설명할 수 있습니다.
