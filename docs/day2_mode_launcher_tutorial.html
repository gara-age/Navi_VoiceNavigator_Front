Day 2 Tutorial

Flutter 2일차: 모드 선택 화면 만들기

이 문서는 2일차에 진행한 mode_launcher_page.dart와 demo_home_page.dart 작업을 클론코딩/튜토리얼 방식으로 다시 읽을 수 있게 정리한 문서입니다.

1단계

단순 버튼 화면을 카드형 모드 선택 화면으로 바꿉니다.

2단계

카드 공통 UI를 _ModeLaunchCard로 분리합니다.

3단계

실제 모드 카드를 HomePage에 연결합니다.

4단계

DemoHomePage를 만들어 데모 모드 진입점을 만듭니다.

5단계

실행해서 두 모드가 모두 정상 이동하는지 확인합니다.

오늘의 목표

  • 앱 첫 화면이 더 이상 버튼 1개짜리 임시 화면이 아니어야 합니다.
  • 실제 모드와 데모 모드 카드가 보여야 합니다.
  • 카드를 누르면 각각 다른 화면으로 이동해야 합니다.
  • 1일차에서 만든 테마/색상 시스템이 실제 UI에 반영되어야 합니다.

먼저 이해할 개념

Scaffold

한 화면의 기본 틀입니다. 앱의 페이지 단위 바탕이라고 생각하면 됩니다.

SafeArea

상단 상태바나 노치에 UI가 겹치지 않게 안전 영역 안에 배치합니다.

ConstrainedBox

데스크탑에서 UI가 너무 넓게 벌어지지 않도록 최대 폭을 제한합니다.

Row + Expanded

카드 2개를 가로로 나란히 두고 균등하게 너비를 나누는 데 사용합니다.

Navigator.push

현재 화면 위에 새로운 화면을 올려 이동하는 Flutter 기본 방식입니다.

1. mode_launcher_page.dart 구조 바꾸기

lib/app/mode_launcher_page.dart

기존 파일은 제목과 버튼 하나만 있는 단순한 구조였습니다. 2일차에는 이 화면을 카드형 모드 선택 화면으로 업그레이드합니다.

핵심 생각: 화면 전체 레이아웃은 하나의 큰 Column이고, 그 안에 카드 두 개를 Row로 배치합니다.

목표 구조

Scaffold
  SafeArea
    Padding
      Center
        ConstrainedBox
          Container
            Column
              앱 아이콘
              앱 제목
              설명 문구
              Row
                실제 모드 카드
                데모 모드 카드
              하단 안내 문구

왜 카드 공통 위젯으로 빼는가

  • 카드 2개는 거의 같은 모양입니다.
  • 제목, 설명, 아이콘, 색상, 이동 화면만 다릅니다.
  • 그래서 _ModeLaunchCard로 빼면 중복이 줄고 유지보수가 쉬워집니다.

2. Navigator로 화면 이동 연결하기

실제 모드 카드는 HomePage로 이동하고, 데모 모드 카드는 DemoHomePage로 이동합니다. 

void _openRealMode(BuildContext context) {
  Navigator.of(context).push(
    MaterialPageRoute<void>(
      builder: (_) => const HomePage(),
    ),
  );
}

void _openDemoMode(BuildContext context) {
  Navigator.of(context).push(
    MaterialPageRoute<void>(
      builder: (_) => const DemoHomePage(),
    ),
  );
}

흐름은 단순합니다. 카드가 눌리면 해당 함수가 실행되고, Navigator.push가 새 화면을 현재 화면 위에 쌓습니다.

3. _ModeLaunchCard 공통 카드 만들기

카드 두 개를 각각 직접 쓰는 대신, 공통 카드 위젯을 하나 만들고 다른 값만 파라미터로 넣습니다.

카드마다 바뀌는 값

  • title
  • subtitle
  • badge
  • hint
  • icon
  • accentColor
  • background
  • onTap

왜 좋은가

  • UI가 통일됩니다.
  • 색상이나 여백을 바꾸고 싶을 때 한 번만 수정하면 됩니다.
  • Flutter의 컴포넌트 분리 감각을 익히게 됩니다.

4. demo_home_page.dart 만들기

lib/demo/demo_home_page.dart

오늘은 데모 로직 자체를 완성하는 날이 아닙니다. 중요한 것은 데모 모드 진입점을 먼저 만드는 것입니다.

그래서 2일차의 DemoHomePage는 아주 단순한 플레이스홀더로 두는 게 맞습니다.

핵심 생각: 오늘은 데모 화면을 완성하는 게 아니라, “데모 모드로 들어가는 구조”를 먼저 만드는 날입니다.

최종 코드 1: mode_launcher_page.dart

import 'package:flutter/material.dart';

import '../demo/demo_home_page.dart';
import '../features/home/presentation/home_page.dart';
import 'theme/colors.dart';

class ModeLauncherPage extends StatelessWidget {

      MaterialPageRoute<void>(
        builder: (_) => const HomePage(),
      ),

  }

    Navigator.of(context).push(
      MaterialPageRoute<void>(
        builder: (_) => const DemoHomePage(),
      ),

  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      body: SafeArea(
        child: Padding(
          padding: const EdgeInsets.all(20),
          child: Center(
            child: ConstrainedBox(
              constraints: const BoxConstraints(maxWidth: 980),
              child: Container(
                padding: const EdgeInsets.all(28),
                decoration: BoxDecoration(
                  color: Colors.white,
                  borderRadius: BorderRadius.circular(24),
                  border: Border.all(color: AppColors.border),
                  boxShadow: const [
                    BoxShadow(
                      color: Color(0x14000000),
                      blurRadius: 24,
                      offset: Offset(0, 10),
                    ),
                  ],
                ),
                child: Column(
                  mainAxisSize: MainAxisSize.min,
                  children: [
                    Container(
                      width: 72,
                      height: 72,
                      decoration: BoxDecoration(
                        color: AppColors.accent,
                        borderRadius: BorderRadius.circular(20),
                      ),
                      child: const Icon(
                        Icons.mic_rounded,
                        color: Colors.white,
                        size: 36,
                      ),
                    ),
                    const SizedBox(height: 18),
                    const Text(
                      'Navi: Voice Navigator',
                      style: TextStyle(
                        fontSize: 28,
                        fontWeight: FontWeight.w800,
                      ),
                    ),
                    const SizedBox(height: 8),
                    const Text(
                      '실행할 모드를 선택하세요.',
                      style: TextStyle(
                        fontSize: 15,
                        color: AppColors.textMuted,
                      ),

                    const SizedBox(height: 28),

                            subtitle: '실제 홈 화면과 연결되는 기본 앱 흐름입니다.',

                            background: AppColors.warningSoft,

                          ),

class _ModeLaunchCard extends StatelessWidget {
  const _ModeLaunchCard({
    required this.title,
    required this.subtitle,
    required this.badge,
    required this.hint,
    required this.icon,
    required this.accentColor,
    required this.background,
    required this.onTap,
  });

  final String title;
  final String subtitle;
  final String badge;
  final String hint;
  final IconData icon;
  final Color accentColor;
  final Color background;
  final VoidCallback onTap;

  @override
  Widget build(BuildContext context) {
    return InkWell(
      onTap: onTap,
      borderRadius: BorderRadius.circular(20),
      child: Container(
        padding: const EdgeInsets.all(20),
        decoration: BoxDecoration(
          color: Colors.white,
          borderRadius: BorderRadius.circular(20),
          border: Border.all(color: AppColors.border),
        ),
        child: Column(
          crossAxisAlignment: CrossAxisAlignment.start,
          children: [
            Container(
              padding: const EdgeInsets.symmetric(horizontal: 10, vertical: 6),
              decoration: BoxDecoration(
                color: background,
                borderRadius: BorderRadius.circular(999),
              ),
              child: Text(
                badge,
                style: TextStyle(
                  fontSize: 12,
                  fontWeight: FontWeight.w700,
                  color: accentColor,
                ),
              ),
            ),
            const SizedBox(height: 18),
            Container(
              width: 56,
              height: 56,
              decoration: BoxDecoration(
                color: background,
                borderRadius: BorderRadius.circular(16),
              ),
              child: Icon(
                icon,
                color: accentColor,
                size: 28,
              ),
            ),
            const SizedBox(height: 18),
            Text(
              title,
              style: const TextStyle(
                fontSize: 20,
                fontWeight: FontWeight.w700,
              ),
            ),
            const SizedBox(height: 8),
            Text(
              subtitle,
              style: const TextStyle(
                fontSize: 14,
                color: AppColors.textMuted,
                height: 1.5,
              ),
            ),
            const SizedBox(height: 18),
            Row(
              children: [
                Text(
                  hint,
                  style: TextStyle(
                    fontSize: 13,
                    fontWeight: FontWeight.w600,
                    color: accentColor,
                  ),
                ),
                const Spacer(),
                Icon(
                  Icons.arrow_forward_rounded,
                  color: accentColor,
                ),
              ],
            ),
          ],
        ),
      ),
    );
  }
}

2일차 완료 기준

화면 기준

  • 앱 첫 화면에 카드 2개가 보입니다.
  • 실제 모드와 데모 모드가 구분됩니다.
  • 더 이상 버튼 하나짜리 임시 화면이 아닙니다.

동작 기준

  • 실제 모드 클릭 시 HomePage로 이동합니다.
  • 데모 모드 클릭 시 DemoHomePage로 이동합니다.
  • 뒤로 가기 시 다시 모드 선택 화면으로 돌아옵니다.

학습 기준

  • Navigator.push를 설명할 수 있습니다.
  • Row + Expanded 배치를 이해합니다.
  • 공통 UI를 별도 위젯으로 빼는 이유를 설명할 수 있습니다.
